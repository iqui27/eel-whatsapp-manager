**Fix the loading vs empty state bug:**

The root cause: `loading={groupsData.length === 0}` means "show loading if there are no groups" — but if there are genuinely no groups, this shows loading forever.

Solution: Track whether initial data has loaded with an `isLoaded` flag.

In `src/app/operacoes/page.tsx`:
1. The page already has `isLoading` state that starts `true` and is set to `false` in `finally` of `fetchOperations`. Use `!isLoading` as the "loaded" signal.
2. Change line 391: `loading={groupsData.length === 0}` to `loading={isLoading}` — this shows loading skeleton only during initial load, then shows empty state if no groups.
3. Change line 406: `loading={messagesData.length === 0}` to `loading={isLoading}` — same fix for messages.

The `GroupCapacityGrid` and `MessageFeed` components already have proper `if (loading)` and `if (data.length === 0)` branches, so passing `loading={isLoading}` instead of `loading={data.length === 0}` fixes the bug without component changes.

**Populate voter count:**
4. In the `fetchOperations` callback, add a fetch to `/api/voters?limit=1` to get the voter total:
   ```typescript
   const voterRes = await fetch('/api/voters?limit=1');
   if (voterRes.ok) {
     const voterData = await voterRes.json();
     setVoterTotal(typeof voterData?.total === 'number' ? voterData.total : 0);
   }
   ```
   Add to the Promise.all array alongside the other fetches.
   
5. Add state: `const [voterTotal, setVoterTotal] = useState(0);`

6. Update the `systemState` object (line 214): Change `voters: { total: 0 }` to `voters: { total: voterTotal }`.

**Add error toasts:**
7. Import `toast` from `sonner` at the top.
8. In `fetchOperations`, after each `.ok` check, add error toasts for failures:
   - `if (!opsRes.ok) toast.error('Erro ao carregar operacoes');`
   - `if (!kpisRes.ok) toast.error('Erro ao carregar KPIs');`
   - `if (!msgsRes.ok) toast.error('Erro ao carregar mensagens');`
   - `if (!groupsRes.ok) toast.error('Erro ao carregar grupos');`
   Only show toasts when `showRefreshing` is true (manual refresh) to avoid spamming on auto-refresh failures.
