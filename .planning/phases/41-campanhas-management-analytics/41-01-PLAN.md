  1. **Activate/Pause buttons in campaign list:**
     - For `draft` campaigns: show "Ativar" button (green) — changes to `scheduled` status
     - For `scheduled` campaigns: show "Pausar" button (orange) — changes to `paused`
     - For `sending` campaigns: show "Pausar" button (orange) — changes to `paused` (stops sending)
     - For `paused` campaigns: show "Retomar" button (green) — changes back to `sending` or `scheduled`
     - For `sent` or `cancelled` campaigns: no activate/pause button (terminal states)
     - Button styles: small, inline with the actions column
     - Confirmation dialog for Pause (uses shadcn AlertDialog): "Tem certeza que deseja pausar a campanha '{name}'?"
     - No confirmation needed for Activate/Resume (reversible actions)

  2. **Status transition validation on backend:**
     In PUT /api/campaigns route.ts, add status transition validation:
     ```
     Valid transitions:
     draft → scheduled (activate)
     draft → cancelled
     scheduled → sending (when cron picks it up, or manual send)
     scheduled → paused
     scheduled → cancelled
     sending → paused
     sending → sent (when all messages delivered)
     paused → scheduled (resume)
     paused → sending (resume if was sending)
     paused → cancelled
     ```
     Invalid transitions should return 400 with error message.

  3. **Optimistic UI updates:**
     - When user clicks Activate/Pause, immediately update the UI state
     - If the API call fails, revert the state and show error toast
     - Pattern: `setCampaigns(prev => prev.map(c => c.id === id ? { ...c, status: newStatus } : c))`

  4. **db-campaigns.ts** — Add a `updateCampaignStatus(id: string, status: string)` function that:
     - Validates the transition is legal
     - Updates status + updatedAt
     - For pause: saves the previous status in a `previousStatus` field (so resume knows where to go back)
     - Returns the updated campaign

  5. **Syslog** — Log all status transitions: `syslog.info('campaign', 'Campaign status changed', { campaignId, from: oldStatus, to: newStatus })`
