Improve SSE reconnection reliability:

1. **Exponential backoff**: The current reconnect logic uses a fixed timer pattern. Verify and if needed update to use exponential backoff: `Math.min(1000 * 2^attempt, 30000)` for reconnect delays (1s, 2s, 4s, 8s, 16s, 30s cap). The hook already has `MAX_RECONNECT_ATTEMPTS = 5` — increase to `8` to give more chances on flaky connections.

2. **Stale connection detection**: Add a heartbeat timeout. If no event (heartbeat or data) arrives within 45 seconds, proactively close the EventSource and trigger reconnect. The server sends heartbeats via `conversation:heartbeat` events. Add a `heartbeatTimeoutRef` that resets on any event and triggers reconnect on expiry:
   ```typescript
   const heartbeatTimeoutRef = useRef<number | null>(null);
   const HEARTBEAT_TIMEOUT_MS = 45000;
   
   const resetHeartbeatTimer = useCallback(() => {
     if (heartbeatTimeoutRef.current) window.clearTimeout(heartbeatTimeoutRef.current);
     heartbeatTimeoutRef.current = window.setTimeout(() => {
       // No events for 45s — connection is stale, force reconnect
       disconnect();
       scheduleReconnect();
     }, HEARTBEAT_TIMEOUT_MS);
   }, [disconnect, scheduleReconnect]);
   ```
   Call `resetHeartbeatTimer()` inside every event handler (onmessage, onopen).

3. **Cleanup on unmount**: Ensure `heartbeatTimeoutRef` is cleared in the cleanup function alongside the existing `eventSourceRef` cleanup.

4. **Online/offline event listeners**: Add `window.addEventListener('online', ...)` to immediately attempt reconnect when browser comes back online, instead of waiting for the next backoff timer. Add `window.addEventListener('offline', ...)` to immediately set status to 'offline' and stop reconnect attempts.

5. **Export `forceReconnect`** from the hook return value so the conversas page can offer a manual "reconnect" button. Add:
   ```typescript
   const forceReconnect = useCallback(() => {
     reconnectAttemptsRef.current = 0;
     disconnect();
     // Will trigger connect on next effect cycle
   }, [disconnect]);
   ```
   Add `forceReconnect` to the return object.
