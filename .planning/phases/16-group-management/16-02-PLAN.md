# Phase 16 Plan 02: Groups Management Page UI + Campaign Integration

## Objective

Build the groups management UI page with group list, create dialog, capacity visualization, invite link management, and campaign integration for automatic invite link inclusion in messages.

## Context

- Backend from 16-01: whatsapp_groups table, db-groups.ts, webhook handler
- Evolution API has all necessary group endpoints
- Need UI at /grupos for operators to manage groups

## Tasks

### Task 1: Create groups API routes
type: auto

**File:** `src/app/api/groups/route.ts`

**Endpoints:**
- `GET` — List groups with optional filters (status, campaignId)
- `POST` — Create group via Evolution API and store in DB

**File:** `src/app/api/groups/[id]/route.ts`

**Endpoints:**
- `GET` — Get group by ID
- `PUT` — Update group metadata (name, description, status)
- `DELETE` — Archive group (soft delete via status='archived')

**Verification:** Routes compile and follow existing patterns.

### Task 6: Create campaign-group integration hook
type: auto

**File:** `src/lib/campaign-groups.ts`

**Function:** `getActiveInviteLink(campaignId: string)`

**Logic:**
1. Find active groups linked to campaign
2. Filter by status='active'
3. Sort by currentSize (prefer groups with more room)
4. Return invite link of best group
5. If all groups are full, trigger overflow (call createOverflowGroup)

**Function:** `assignGroupToCampaign(groupId: string, campaignId: string)`

**Logic:**
1. Update group's campaignId
2. Add segmentTag if campaign has segment

**Verification:** Functions compile and work with test data.

---

### Task 7: Update message template variable resolver
type: auto

**File:** `src/lib/campaign-variables.ts`

**Add variable:** `{link_grupo}` or `{invite_link}`

**Resolution:**
1. Check if campaign has linked groups
2. Call `getActiveInviteLink(campaignId)`
3. Return invite URL or empty string if no group

**Update:** `hydrateCampaignToQueue()` in db-campaigns.ts to resolve `{link_grupo}`

**Verification:** Variable resolves to actual invite link.

---

### Task 8: Add overflow detection to send-queue cron
type: auto

**File:** `src/app/api/cron/send-queue/route.ts`

**Add logic:**
1. After sending each message, check if it contained invite link
2. If yes, increment group's currentSize estimate
3. If group near capacity (90%), log warning
4. If group full, update status and potentially create overflow

**Note:** This is a soft estimate; real count comes from webhook events.

**Verification:** Logic added without breaking existing flow.
