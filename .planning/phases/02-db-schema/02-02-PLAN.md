**conversations table:**
```typescript
export const conversations = pgTable('conversations', {
  id: uuid('id').primaryKey().default(sql`gen_random_uuid()`),
  voterId: uuid('voter_id').references(() => voters.id, { onDelete: 'set null' }),
  voterName: text('voter_name').notNull(),
  voterPhone: text('voter_phone').notNull(),
  status: text('status', { enum: ['open', 'assigned', 'waiting', 'resolved', 'bot'] }).default('bot'),
  assignedAgent: text('assigned_agent'),       // User ID of assigned human agent
  handoffReason: text('handoff_reason'),       // Why bot handed off to human
  lastMessageAt: timestamp('last_message_at', { withTimezone: true }),
  priority: integer('priority').default(0),    // Higher = more urgent
  createdAt: timestamp('created_at', { withTimezone: true }).default(sql`now()`),
  updatedAt: timestamp('updated_at', { withTimezone: true }).default(sql`now()`),
}, (t) => [
  index('idx_conversations_voter').on(t.voterId),
  index('idx_conversations_status').on(t.status),
  index('idx_conversations_priority').on(t.priority),
]);
```

**conversation_messages table** (separate table, not JSON — better for querying):
```typescript
export const conversationMessages = pgTable('conversation_messages', {
  id: uuid('id').primaryKey().default(sql`gen_random_uuid()`),
  conversationId: uuid('conversation_id').notNull().references(() => conversations.id, { onDelete: 'cascade' }),
  sender: text('sender', { enum: ['voter', 'bot', 'agent'] }).notNull(),
  content: text('content').notNull(),
  createdAt: timestamp('created_at', { withTimezone: true }).default(sql`now()`),
}, (t) => [
  index('idx_conv_messages_conv').on(t.conversationId),
]);
```

**consent_logs table:**
```typescript
export const consentLogs = pgTable('consent_logs', {
  id: uuid('id').primaryKey().default(sql`gen_random_uuid()`),
  voterId: uuid('voter_id').references(() => voters.id, { onDelete: 'cascade' }),
  action: text('action', { enum: ['opt_in', 'opt_out', 'renew', 'revoke'] }).notNull(),
  channel: text('channel').default('whatsapp'),  // Where consent was given
  ipAddress: text('ip_address'),
  metadata: text('metadata'),    // JSON string with extra context
  createdAt: timestamp('created_at', { withTimezone: true }).default(sql`now()`),
}, (t) => [
  index('idx_consent_voter').on(t.voterId),
]);
```

**users table** (app users, not voters):
```typescript
export const users = pgTable('users', {
  id: uuid('id').primaryKey().default(sql`gen_random_uuid()`),
  name: text('name').notNull(),
  email: text('email').notNull(),
  role: text('role', { enum: ['coordenador', 'cabo', 'voluntario', 'admin'] }).default('voluntario'),
  regionScope: text('region_scope'),    // Which region this user can access (null = all)
  enabled: boolean('enabled').default(true),
  createdAt: timestamp('created_at', { withTimezone: true }).default(sql`now()`),
  updatedAt: timestamp('updated_at', { withTimezone: true }).default(sql`now()`),
}, (t) => [
  index('idx_users_email').on(t.email),
  index('idx_users_role').on(t.role),
]);
```

**Part A: Create 3 CRUD libraries following existing patterns (see db-chips.ts for reference):**

**src/lib/db-conversations.ts:**
- `loadConversations(status?: string)` — load all or filter by status, order by priority desc then lastMessageAt desc
- `getConversation(id: string)` — single conversation by id
- `addConversation(data: NewConversation)` — create new conversation
- `updateConversationStatus(id: string, status: string, assignedAgent?: string)` — update status + optional agent
- `addMessage(conversationId: string, sender: string, content: string)` — insert into conversationMessages, update conversation.lastMessageAt
- `getMessages(conversationId: string)` — load messages for a conversation, ordered by createdAt asc

**src/lib/db-compliance.ts:**
- `logConsent(voterId: string, action: string, channel?: string, metadata?: string)` — insert consent log, also update voter.optInStatus accordingly
- `getConsentHistory(voterId: string)` — load consent logs for a voter, ordered by createdAt desc
- `getConsentStats()` — return counts by opt_in_status (active, expired, revoked, pending) using SQL aggregation

**src/lib/db-users.ts:**
- `loadUsers()` — load all users ordered by createdAt desc
- `getUser(id: string)` — single user by id
- `addUser(data: NewUser)` — insert user
- `updateUser(id: string, data: Partial<NewUser>)` — update with updatedAt
- `deleteUser(id: string)` — delete user

**Part B: Create 3 API route files following the pattern in `src/app/api/chips/route.ts`:**

Each route file should:
1. Import `verifyAuth` pattern (check cookie → validateSession → load config) — copy from chips/route.ts
2. Handle GET (list), POST (create), PUT (update), DELETE (delete) methods
3. Return proper JSON responses with try/catch error handling
4. Use NextResponse from 'next/server'

**src/app/api/voters/route.ts:**
- GET: loadVoters() — optional `?search=` query param calls searchVoters()
- POST: addVoter(body) — requires name, phone at minimum
- PUT: updateVoter(body.id, body) — requires id
- DELETE: deleteVoter(searchParams.get('id')) — requires id in query

**src/app/api/campaigns/route.ts:**
- GET: loadCampaigns() — optional `?status=` filter
- POST: addCampaign(body) — requires name, template
- PUT: updateCampaign(body.id, body) — requires id
- DELETE: deleteCampaign(searchParams.get('id'))

**src/app/api/segments/route.ts:**
- GET: loadSegments()
- POST: addSegment(body) — requires name, filters
- PUT: updateSegment(body.id, body) — requires id
- DELETE: deleteSegment(searchParams.get('id'))

All API routes must be auth-protected (same pattern as chips/route.ts — if verifyAuth fails, return 401).
