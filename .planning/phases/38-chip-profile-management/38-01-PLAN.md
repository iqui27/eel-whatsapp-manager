     async function getProfileStatus(instanceName: string): Promise<string | null>

  1. **Chips API profile update** — In the PUT handler of src/app/api/chips/route.ts, add support for `action='updateProfile'`:
     ```
     PUT /api/chips
     Body: { id, action: 'updateProfile', profileName?: string, profilePictureUrl?: string }
     ```
     When action='updateProfile':
     - If profileName provided, call `setProfileName(instanceName, profileName)` via Evolution API
     - If profilePictureUrl provided, call `setProfilePicture(instanceName, profilePictureUrl)` via Evolution API
     - Persist both fields to DB via `updateChip(id, { profileName, profilePictureUrl })`
     - Return updated chip record
     - Log via syslog: `syslog.info('chip', 'Profile updated for chip', { chipId, profileName })`

  2. **Profile sync in health cron** — In the chip-health cron (`/api/cron/chip-health/route.ts`), after checking connection status for each chip, also sync the profile:
     - Call `getProfilePicture(instanceName)` to get current profile picture URL
     - If the URL differs from what's stored in DB, update it
     - This ensures profile pictures stay in sync even if changed outside the app
     - Only sync profile for chips in 'healthy' or 'degraded' state (skip disconnected)
     - Rate limit: only sync profile once per hour (check chip.updatedAt or add a lastProfileSync timestamp)

  Handle all Evolution API errors gracefully — profile operations should never crash the health cron. Wrap in try/catch with syslog.warn.
