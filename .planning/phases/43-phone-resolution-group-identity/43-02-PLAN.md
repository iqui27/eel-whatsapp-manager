<objective>
Surface voter names for group members that WhatsApp hides behind @lid JIDs by querying the group_sender_cache built in Plan 01. Also verify and harden the GROUP_PARTICIPANTS_UPDATE opt-in path so that real-JID participants who join closed groups via invite link also populate the sender cache (making their names visible in the members list).

Purpose: The group members API currently returns `voterName: null` for any @lid participant because @lid numeric IDs are not phone numbers and cannot be mapped directly. However, the sender cache accumulated in Plan 01 contains real phone→groupJid associations from group activity. By querying the cache and then running findVoterByPhone on the resolved phone, we can surface voter names for members who have previously sent a message (or who joined via an invite link that was tracked). Members who have never sent and were not seen via GROUP_PARTICIPANTS_UPDATE stay as "Membro desconhecido" — a documented WhatsApp limitation.

Output: Members API with cache-assisted @lid enrichment. GROUP_PARTICIPANTS_UPDATE writes joining participants to group_sender_cache.
</objective>

```ts
// Used to resolve @lid participants to voter names.
const senderCache = await getGroupSendersByGroupJid(group.groupJid);
// Build phone→voterName map from the cache using findVoterByPhone for dual-format support
const cachePhoneToName: Record<string, string> = {};
for (const entry of senderCache) {
  const voter = await findVoterByPhone(entry.normalizedPhone);
  if (voter) {
    cachePhoneToName[entry.normalizedPhone] = voter.name;
    // Also store the alternative format (12↔13 digit) as a convenience key
    if (entry.normalizedPhone.length === 12) {
      const with9 = entry.normalizedPhone.slice(0, 4) + '9' + entry.normalizedPhone.slice(4);
      cachePhoneToName[with9] = voter.name;
    } else if (entry.normalizedPhone.length === 13) {
      const without9 = entry.normalizedPhone.slice(0, 4) + entry.normalizedPhone.slice(5);
      cachePhoneToName[without9] = voter.name;
    }
  }
}
```
