1. **Create `src/app/perfil/page.tsx`** — a 'use client' page wrapped in SidebarLayout:

2. **Load session data** on mount via `fetch('/api/auth/session')`. The response shape is:
   ```ts
   interface SessionResponse {
     actor: {
       userId: string | null;
       name: string;
       email: string | null;
       role: 'admin' | 'coordenador' | 'cabo' | 'voluntario';
       regionScope: string | null;
       permissions: string[];
       enabled: boolean;
       source: 'user' | 'bootstrap';
       label: string;
     };
     permissions: string[];
     availablePermissions: string[];
     pageAccess: Record<string, boolean>;
   }
   ```

3. **Page layout** — two columns on desktop (lg:grid-cols-2), single column on mobile:
   - **Left column — Profile Info Card**:
     - Avatar placeholder (circle with initials derived from name)
     - Display name (editable text input)
     - Email (read-only, shown as `text-muted-foreground` with a lock icon)
     - Role badge (colored: admin=purple, coordenador=blue, cabo=green, voluntario=gray)
     - Region scope (if not null, show as muted text)
     - Last login: show `createdAt` from session or just "Sessao ativa" with a green dot
     - Save button for name changes (POST to a new endpoint or existing user update)
   
   - **Right column — two cards stacked**:
     - **Password Change Card**:
       - Current password field
       - New password field
       - Confirm new password field
       - Client-side validation: new password min 4 chars, new === confirm
       - Submit via POST to `/api/auth/change-password` (create simple endpoint or note in action that executor should create it)
       - Show success toast on change, error toast on failure
     
     - **Permissions Summary Card**:
       - List the user's permissions grouped by category:
         - Dashboard: dashboard.view
         - CRM: crm.view, crm.edit
         - Campanhas: campaigns.view, campaigns.manage
         - Conversas: conversations.view, conversations.reply
         - etc.
       - Use green check icon for granted, muted dash for not granted
       - Read-only — just informational

4. **Use theme tokens only** — bg-background, bg-card, text-foreground, text-muted-foreground, border-border, bg-primary, etc. ZERO hardcoded hex colors.

5. **Use shadcn components**: Card, CardHeader, CardTitle, CardContent, CardDescription. Use Badge for role display.

6. **Loading state**: Show skeleton cards while session data loads.

7. **Error state**: If session fetch returns 401, redirect to /login.

8. **Import pattern**: 
   ```tsx

   import { Card, CardHeader, CardTitle, CardContent, CardDescription } from '@/components/ui/card';
   import { Badge } from '@/components/ui/badge';
   ```
