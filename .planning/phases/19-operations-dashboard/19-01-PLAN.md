# Phase 19 Plan 01: Chip Health Grid + Campaign Progress + Alerts

## Objective

Build the chip health overview section, active campaigns send progress bars, and alerts panel for the operations dashboard.

## Context

- Chip health data from Phase 14 (8-state health monitoring)
- Campaign progress from Phase 15 (message queue)
- Need real-time operations view

## Tasks

### Task 1: Create chip health grid component
type: auto

**File:** `src/components/chip-health-grid.tsx`

**Display:**
- Grid of chip cards (responsive: 2-4 columns)
- Each card shows:
  - Status indicator (🟢 healthy / 🟡 degraded / 🔴 quarantined)
  - Chip name and phone
  - Progress bar: messages sent today vs daily limit
  - Connection quality
  - Last activity timestamp
  - One-click restart button

**Verification:** Component renders with mock data.

---

### Task 2: Create campaign progress component
type: auto

**File:** `src/components/campaign-progress-bars.tsx`

**Display:**
- List of active/sending campaigns
- Each row shows:
  - Campaign name
  - Progress bar with segments (queued/sending/delivered/failed)
  - Count labels
  - ETA calculation (based on send rate)
  - Status badge

**Verification:** Component renders with mock data.

---

### Task 3: Create alerts panel component
type: auto

**File:** `src/components/alerts-panel.tsx`

**Display:**
- List of active alerts (most recent first)
- Alert types:
  - 🔴 Chip disconnected
  - 🟡 Chip near daily limit (>80%)
  - 🟡 Group near capacity (>90%)
  - 🔴 High block rate (>5%)
  - 🟡 Campaign stalled (>1hr no activity)
- Dismiss button per alert
- Badge with alert count

**Verification:** Component renders with mock data.
