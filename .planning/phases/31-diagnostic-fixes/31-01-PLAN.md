  <name>Task 1: Fix CRM inline edit PUT + table overflow + tag wrapping</name>
  <files>src/app/crm/page.tsx</files>
  <action>
  **Bug 1 — Inline edit PUT missing `id` (CRITICAL):**
  At line 637, the inline edit sends `JSON.stringify(editForm)` but the API at `/api/voters` PUT (line 76) requires `body.id`. The editForm only contains the editable fields (name, phone, zone, section, etc.) but NOT the voter ID.

  Fix: Change line 637 from:
  ```
  body: JSON.stringify(editForm),
  ```
  to:
  ```
  body: JSON.stringify({ id: voter.id, ...editForm }),
  ```

  Also change the fetch URL from `/api/voters?id=${voter.id}` (line 634) to just `/api/voters` — the PUT handler reads `id` from the body, not from query params. The `?id=` param is only used by GET.

  **Bug 2 — Table outer div overflow-hidden:**
  At line 465, the outer `<div>` has `overflow-hidden` which prevents horizontal scrolling on narrow screens. Change `overflow-hidden` to `overflow-x-auto` so the table can scroll horizontally when columns exceed container width.

  **Bug 3 — Tags whitespace conflict:**
  The shadcn `TableCell` component applies `whitespace-nowrap` globally. Tags use `flex-wrap` to wrap, but `whitespace-nowrap` prevents line breaks. Add `whitespace-normal` class to the TableCell that renders tags. Find the TableCell containing the tags Badge mapping and add `className="whitespace-normal"` to it (or append to existing className).
  </action>
