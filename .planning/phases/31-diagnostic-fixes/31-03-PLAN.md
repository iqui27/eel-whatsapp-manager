  The POST handler creates the group and stores `admins` in the DB record (line 120), but NEVER calls the Evolution API to actually promote those phone numbers to group admins.

  After the group is created and the invite code is fetched (around line 106, after the try/catch for invite code), add admin promotion:

  1. Import `updateParticipant` from `@/lib/evolution`
  2. After `inviteCode` is obtained (or attempted), if `admins` array has entries, loop through and promote each:
     ```ts
     // Promote admins in WhatsApp group
     if (admins && admins.length > 0) {
       for (const adminPhone of admins) {
         try {
           await updateParticipant(
             config.evolutionApiUrl,
             config.evolutionApiKey,
             instanceName,
             created.id,  // groupJid
             'promote',
             [adminPhone]
           );
           console.log('[api/groups] Promoted admin:', adminPhone);
         } catch (promoteErr) {
           console.warn('[api/groups] Failed to promote admin:', adminPhone, promoteErr);
           // Continue with other admins — don't fail the whole creation
         }
       }
     }
     ```
  3. Place this BEFORE the `createGroupRecord()` call so that if promotion partially fails, the group is still saved.
  4. The `updateParticipant` function signature from `evolution.ts` is: `updateParticipant(apiUrl, apiKey, instanceName, groupJid, action, participants)` where action is 'add'|'remove'|'promote'|'demote' and participants is string[].
