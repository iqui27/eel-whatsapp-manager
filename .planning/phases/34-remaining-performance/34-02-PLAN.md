  <action>
Add an in-memory connection tracker at module scope in `src/app/api/conversations/stream/route.ts`:

```typescript
// ─── Connection Tracking ──────────────────────────────────────────────────────
const MAX_CONNECTIONS_GLOBAL = 50;
const MAX_CONNECTIONS_PER_USER = 3;
const MAX_CONNECTION_LIFETIME_MS = 5 * 60 * 1000; // 5 minutes — forces reconnect (client already handles this)

interface ActiveConnection {
  userId: string;
  startedAt: number;
}

const activeConnections = new Map<string, ActiveConnection>(); // connectionId → info
let connectionCounter = 0;
```

Modify the GET handler:

1. **After auth check, before creating stream:** Extract `userId` from `auth.actor.userId` (or `auth.actor.id`). Generate a unique `connectionId` = `${userId}-${++connectionCounter}`.

2. **Check global limit:** `if (activeConnections.size >= MAX_CONNECTIONS_GLOBAL)` → return 429 with `{ error: 'Limite global de conexões atingido', limit: MAX_CONNECTIONS_GLOBAL }`

3. **Check per-user limit:** Count entries where `conn.userId === userId`. If >= MAX_CONNECTIONS_PER_USER → return 429 with `{ error: 'Limite de conexões por usuário atingido', limit: MAX_CONNECTIONS_PER_USER, current: count }`

4. **Register connection:** `activeConnections.set(connectionId, { userId, startedAt: Date.now() })`

5. **In the stream's `close()` function:** Add `activeConnections.delete(connectionId)` to remove the connection when it ends.

6. **Add max lifetime:** Inside the `while` loop, after `waitForNextPoll`, add:
   ```typescript
   // Server-side max lifetime — forces reconnect to prevent stale connections
   if (Date.now() - connectionStartedAt > MAX_CONNECTION_LIFETIME_MS) {
     close();
     break;
   }
   ```
   The client hook (`use-conversation-stream.ts`) already has reconnect logic, so this is seamless.

7. **Add a periodic stale connection cleanup** (safety valve): Every poll cycle, if `cycleCount % 60 === 0` (every 5 minutes), iterate `activeConnections` and delete entries where `Date.now() - conn.startedAt > MAX_CONNECTION_LIFETIME_MS * 2` (10 minutes — double the lifetime as safety margin for connections that didn't clean up properly).

Do NOT create a separate file for the tracker — keep it in this route file since it's module-scoped and specific to this endpoint.
