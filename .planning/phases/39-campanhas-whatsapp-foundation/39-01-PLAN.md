  <name>Task 1: Create WhatsApp text formatting parser</name>
  <files>src/lib/whatsapp-format.ts</files>
  <action>
  Create src/lib/whatsapp-format.ts — a pure function library that parses WhatsApp-style text formatting into React-renderable segments:

  **WhatsApp formatting rules:**
  - `*text*` → **bold** (rendered with `<strong>`)
  - `_text_` → _italic_ (rendered with `<em>`)
  - `~text~` → ~~strikethrough~~ (rendered with `<del>`)
  - `` `text` `` → `monospace` (rendered with `<code>`)
  - ` ```text``` ` → code block (rendered with `<pre><code>`)
  - URLs (http:// or https://) → clickable links (rendered with `<a>` styled in blue/underline)
  - `\n` → line breaks

  **Implementation approach:**
  ```typescript
  // Input: raw WhatsApp text string
  // Output: array of segments with type and content
  
  type FormatSegment = 
    | { type: 'text'; content: string }
    | { type: 'bold'; content: string }
    | { type: 'italic'; content: string }
    | { type: 'strikethrough'; content: string }
    | { type: 'monospace'; content: string }
    | { type: 'codeblock'; content: string }
    | { type: 'link'; url: string; display: string }
    | { type: 'linebreak' }

  export function parseWhatsAppFormat(text: string): FormatSegment[]
  ```

  **Rules:**
  - Formatting markers must be at word boundaries (not mid-word)
  - Nested formatting is NOT supported in WhatsApp (keep it simple)
  - URLs are detected via regex: `https?://[^\s]+`
  - Empty format markers (e.g., `**`) should be treated as literal text
  - Handle edge cases: unclosed markers, markers at end of text, consecutive markers

  **Also export:**
  ```typescript
  // Strip all formatting markers for plain text (useful for char counting)
  export function stripWhatsAppFormat(text: string): string

  // Count characters excluding format markers
  export function countWhatsAppChars(text: string): number
  ```

  This is a pure utility — no React imports, no 'use client'. Fully testable.
