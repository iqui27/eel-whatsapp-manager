    2. Update POST handler (chip creation) to accept proxy fields from the request body and save them to the chips table.

    3. Update PUT handler (chip update) to accept proxy fields. When proxy config changes, also call `updateInstanceSettings()` to propagate to Evolution API.

    **evolution.ts:**
    1. Update `CreateInstanceOptions` interface to include optional proxy fields:
       ```ts
       proxyHost?: string;
       proxyPort?: number;
       proxyProtocol?: 'http' | 'https' | 'socks4' | 'socks5';
       proxyUsername?: string;
       proxyPassword?: string;
       ```

    2. Update `createInstance()` to include proxy in the request body when provided:
       ```ts
       if (options.proxyHost) {
         body.proxy = {
           host: options.proxyHost,
           port: String(options.proxyPort ?? 80),
           protocol: options.proxyProtocol ?? 'http',
           username: options.proxyUsername ?? '',
           password: options.proxyPassword ?? '',
         };
       }
       ```
       Note: Evolution API v2 accepts proxy in the instance creation body.

    3. Add or update an `updateInstanceSettings()` function that can push proxy changes to a running instance. If such a function doesn't exist, add one that calls `PUT /instance/update/{instanceName}` with the proxy object. If the Evolution API doesn't support runtime proxy update (check docs), document this limitation in a code comment and note that proxy only takes effect on instance creation/restart.

    4. Set `alwaysOnline: false` in `createInstance()` (currently `true` — Evolution API best practice for anti-ban is to disable always-online presence).
