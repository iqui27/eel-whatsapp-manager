  <action>
    Three fixes in the system logger:

    1. **Change default SYSLOG_MIN_LEVEL from 'warn' to 'info'** in `src/lib/system-logger.ts` line 49:
       ```ts
       return env && env in LEVEL_ORDER ? env : 'info';
       ```
       This means info-level logs now get persisted to DB by default, making cron activity visible.

    2. **Add console.error to the flushBuffer catch block** (line 78-80) so DB insert errors are not silently swallowed:
       ```ts
       } catch (err) {
         console.error('[syslog] batch flush failed:', err instanceof Error ? err.message : err);
       }
       ```

    3. **Instrument all 8 cron routes** with syslog calls. For each cron file:
       - Add `import { syslog } from '@/lib/system-logger';` at top
       - Add an info-level log at the START of the cron: `syslog({ level: 'info', category: 'cron', message: '{cron-name} started' })`
       - Add an info-level log at the END with summary: `syslog({ level: 'info', category: 'cron', message: '{cron-name} completed', details: { ...summary } })`
       - Add a warn/error-level log on failures: `syslog({ level: 'error', category: 'cron', message: '{cron-name} failed', details: { error: err.message } })`

    Cron-specific instrumentation details:
    - **send-queue**: Log batch size, messages sent, failures, chips used
    - **campaigns**: Log campaigns dispatched, campaigns skipped (wrong status/time)
    - **chip-health**: Log chips checked, restarts triggered, status changes
    - **reset-counters**: Log chips reset, previous totals
    - **warming**: Log messages sent, chips warmed
    - **ai-profile**: Log voters analyzed, tiers assigned
    - **group-overflow**: Log groups checked, overflows created
    - **reports**: Log schedules checked, reports dispatched

    Keep syslog calls minimal — one start, one end, one on error. Do NOT add debug-level noise.
