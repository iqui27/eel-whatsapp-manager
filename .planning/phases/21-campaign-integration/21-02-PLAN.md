# Plan 21-02: Campaign Queue Integration with Group Links

## Objective
Fazer a hidratação da fila resolver automaticamente o `{link_grupo}` baseado no segmento da campanha.

## Context
Quando uma campanha é criada, as mensagens vão para a fila. Cada mensagem precisa ter o `{link_grupo}` resolvido para o grupo correto do segmento.

## Tasks

### Task 1: Create `resolveCampaignVariables()` function
type: auto

**Implementation:**
1. Create function in `src/lib/campaign-groups.ts`
2. Accept: template, voter, segmentTag, campaignId
3. Resolve all variables including `{link_grupo}`
4. Cache resolved links for campaign duration
5. Handle missing group (create or throw error)

**Files:**
- `src/lib/campaign-groups.ts`

---

### Task 2: Update `hydrateCampaignToQueue()` function
type: auto

**Implementation:**
1. Get segment from campaign
2. Get group for segment using `getGroupForSegment()`
3. If no group, create one with `getOrCreateGroupForSegment()`
4. Resolve template for each voter
5. Store resolved message in queue

**Files:**
- `src/lib/db-message-queue.ts`

---

### Task 3: Add group link caching
type: auto

**Implementation:**
1. Cache group links in memory during campaign
2. Avoid repeated lookups for same segment
3. Invalidate cache when group status changes
4. Store in `campaignDeliveryEvents` for audit

**Files:**
- `src/lib/group-link-cache.ts` (new file)

---

### Task 4: Handle missing group scenarios
type: auto

**Implementation:**
1. If segment has no group, auto-create
2. If segment has no chip assigned, use best available
3. If no chips available, queue as "waiting_chip"
4. Log warning for manual intervention

**Files:**
- `src/lib/db-message-queue.ts`

---

### Task 5: Update webhook handlers for campaign context
type: auto

**Implementation:**
1. Store campaignId in outbound messages metadata
2. Correlate replies to campaign
3. Track conversion per campaign/group

**Files:**
- `src/app/api/webhook/route.ts`

---

## Success Criteria

- [ ] `{link_grupo}` resolves to correct group for segment
- [ ] Group created automatically if missing
- [ ] Cache prevents repeated lookups
- [ ] Webhooks track campaign context
- [ ] All variables resolve correctly
