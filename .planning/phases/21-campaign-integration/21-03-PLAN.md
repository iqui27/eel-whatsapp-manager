# Plan 21-03: Automatic Group Overflow

## Objective
Detectar quando grupo está cheio e criar overflow automaticamente, atualizando o link da campanha.

## Tasks

### Task 1: Create overflow detection cron
type: auto

**Implementation:**
1. Create `/api/cron/group-overflow` endpoint
2. Check all active groups for capacity (>= 90%)
3. Mark groups needing overflow
4. Create overflow groups automatically
5. Update campaign links

**Files:**
- `src/app/api/cron/group-overflow/route.ts`

---

### Task 2: Implement `createOverflowGroupIfNeeded()`
type: auto

**Implementation:**
1. Check if group needs overflow
2. Get segment and chip from original group
3. Create new group with same settings
4. Add suffix (2), (3), etc.
5. Return new group with invite link

**Files:**
- `src/lib/db-groups.ts`

---

### Task 3: Update campaign to use new group link
type: auto

**Implementation:**
1. When overflow created, update cached links
2. New queue messages use new link
3. Invalidate old link cache
4. Log overflow event

**Files:**
- `src/lib/group-link-cache.ts`

---

### Task 4: Add notification for overflow events
type: auto

**Implementation:**
1. Create notification helper
2. Send alert when overflow triggered
3. Show in dashboard alerts panel
4. Optional: webhook for external systems

**Files:**
- `src/lib/notifications.ts` (new file)

---

### Task 5: Update dashboard alerts
type: auto

**Implementation:**
1. Add overflow warning to alerts panel
2. Show group name and capacity
3. Link to group management
4. Show "overflow created" success message

**Files:**
- `src/components/alerts-panel.tsx`

---

## Success Criteria

- [ ] Overflow cron detects groups at 90%+
- [ ] New group created automatically
