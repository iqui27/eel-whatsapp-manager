  Rewrite `src/app/api/webhook/route.ts` to replace the broken old imports:

  **Remove:**
  - `import { loadConfig } from '@/lib/config'` → use `import { loadConfig } from '@/lib/db-config'`
  - `import { loadChips, updateChip } from '@/lib/chips'` → use `import { loadChipsWithClusters } from '@/lib/db-chips'` and `db.update(chips)` from Drizzle directly for status updates (or add an `updateChipByName` helper to db-chips.ts if it doesn't have one)

  **For `connection.update` event:**
  - Load chips via `loadChipsWithClusters()` (or a lighter `loadChips()` if available in db-chips.ts)
  - Find chip by `instanceName` matching `chip.name`
  - Update chip status using Drizzle: `db.update(chips).set({ status: newStatus }).where(eq(chips.id, chip.id))`
  - Import `chips` table from `@/db/schema` and `eq` from `drizzle-orm`

  **For `messages.upsert` event:**
  - For each incoming message where `msg.key?.fromMe === false`:
    - Extract `remoteJid` (sender phone), strip `@s.whatsapp.net` suffix to get clean phone number
    - Extract message text from `msg.message?.conversation || msg.message?.extendedTextMessage?.text || ''`
    - Extract `instanceName` from the webhook body to know which chip received it
    - Look up voter by phone: `searchVoters(phone)` from `db-voters.ts` — if found, get voterId, voterName
    - Check if an open conversation exists for this phone: query `conversations` table where `voterPhone = phone AND status IN ('open', 'pending')`
    - If conversation exists: add message via `addMessage(conversationId, 'voter', messageText)`
    - If no conversation: create one via `addConversation({ voterId, voterName, voterPhone: phone, status: 'pending', chipId: instanceName })` then `addMessage(newConv.id, 'voter', messageText)`
  - Import `addConversation`, `addMessage` from `@/lib/db-conversations`
  - Import `searchVoters` from `@/lib/db-voters`

  **Keep GET handler** as-is (returns `{ status: 'ok' }`).

  **Do NOT** add auth to the webhook route — Evolution API calls it without auth tokens.
