<task type="auto">
  <name>Task 1: Implement real campaign send via Evolution API</name>
  <files>src/app/api/campaigns/[id]/send/route.ts</files>
  <action>
  Rewrite the send route to actually deliver WhatsApp messages:

  **Request body:** Accept `{ chipId?: string }` — the chip instance name to use for sending. If not provided, pick the first connected chip from DB.

  **Send pipeline:**
  1. Fetch campaign from DB (already done)
  2. Get config via `loadConfig()` from `db-config.ts` to get Evolution API URL + API key
  3. Get segment voters: `getSegmentVoterIds(campaign.segmentId)` from `db-segments.ts`
  4. If segmentId is null/empty, return 400: "Campanha sem segmento definido"
  5. Fetch full voter records for those IDs: query `voters` table with `inArray(voters.id, voterIds)`
  6. Determine chip: use `chipId` from request body, or query `chips` table for first chip with `status = 'connected'`
  7. If no connected chip found, return 400: "Nenhum chip conectado disponível"
  8. Set campaign status to `'sending'` with `totalSent: voterIds.length`
  9. **Send loop (async, non-blocking):** Use an async IIFE that runs after the response is sent:
     - For each voter: template the message by replacing `{nome}` with voter.name, `{bairro}` with voter.neighborhood (or empty), `{zona}` with voter.zone, etc.
     - Call `sendText(config.evolutionApiUrl, config.evolutionApiKey, chipInstanceName, voter.phone, templatedMessage)`
     - Track: `sent++`, on success `delivered++`, on error `failed++`
     - Add a small delay between sends: `await new Promise(r => setTimeout(r, 200))` — to avoid rate limiting
     - After all sends, update campaign: `status: 'sent'`, `totalSent: sent`, `totalDelivered: delivered`, `totalFailed: failed`
  10. Return immediately: `{ campaignId: id, status: 'sending', audience: voterIds.length }`

  **Import** `sendText` from `@/lib/evolution`, `getSegmentVoterIds` from `@/lib/db-segments`, `loadConfig` from `@/lib/db-config`, `voters, chips` from `@/db/schema`, `inArray` from `drizzle-orm`.

  **Do NOT** block the HTTP response on sending — the actual send loop runs asynchronously after the response is returned. The monitor page polls campaign status to show progress.

  **Variable replacement:** Create a simple `templateMessage(template: string, voter: Voter)` helper function in the same file that replaces `{nome}`, `{bairro}`, `{zona}`, `{secao}`, `{interesse}` with voter fields. Unknown variables are left as-is.
  </action>
