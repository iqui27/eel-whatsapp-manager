<task type="auto">
  <name>Task 1: Add ?id= and pagination to voters API</name>
  <files>src/app/api/voters/route.ts</files>
  <action>
  Update the GET handler in `/api/voters/route.ts`:

  **ID lookup:**
  1. Parse `?id=` from searchParams
  2. If `id` is present: call `getVoter(id)` from `db-voters.ts` (it already exists)
  3. Return the single voter or 404
  4. Priority: check `id` first, then `search`, then list all

  **Pagination:**
  1. Parse `?page=` (default 1) and `?limit=` (default 50) from searchParams
  2. For the list-all case (no id, no search): use `db.select().from(voters).orderBy(desc(voters.createdAt)).limit(limit).offset((page - 1) * limit)`
  3. Also get total count: `db.select({ count: count() }).from(voters)`
  4. Return `{ data: voters[], total: number, page: number, limit: number }`
  5. For search results, also apply pagination similarly
  6. Import `count, desc` from `drizzle-orm`, `getVoter` from `db-voters.ts`

  **IMPORTANT:** The existing non-paginated response was `NextResponse.json(data)` returning a flat array. The new paginated response returns an object `{ data, total, page, limit }`. The CRM page must be updated to handle this new format (Task 2).
  </action>

  </verify>
  <done>GET /api/voters?id=X returns single voter, GET /api/voters?page=1&limit=50 returns paginated results with total count</done>
