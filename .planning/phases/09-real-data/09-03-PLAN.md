  <files>src/app/api/conversations/[id]/messages/route.ts, src/app/api/conversations/route.ts</files>
  <action>
  **Messages POST — send via WhatsApp when sender is 'agent':**
  1. In the POST handler of `messages/route.ts`, after validating body:
  2. If `body.sender === 'agent'`:
     - Fetch the conversation to get `voterPhone` and the chip used (if stored) or default chip
     - Load config via `loadConfig()` from `db-config.ts` to get `evolutionApiUrl` and `evolutionApiKey`
     - Determine chip instance name: if conversation has a `chipId`, use it. Otherwise, query chips table for first connected chip
     - Call `sendText(config.evolutionApiUrl, config.evolutionApiKey, chipInstanceName, conversation.voterPhone, body.content)`
     - If sendText throws, return 500 with error message (don't store the message if it fails to send)
     - If sendText succeeds, then call `addMessage(id, 'agent', body.content)` as before
  3. If `body.sender !== 'agent'` (e.g. 'voter' or 'bot'), just store the message without sending (inbound messages are received via webhook, not sent)
  4. Import `sendText` from `@/lib/evolution`, `loadConfig` from `@/lib/db-config`, `getConversation` from `@/lib/db-conversations`

  **Conversations DELETE handler (route.ts):**
  1. Add `export async function DELETE(request: NextRequest)` to `/api/conversations/route.ts`
  2. Parse `?id=` from searchParams
  3. Auth check via `verifyAuth`
  4. Delete conversation and its messages: `db.delete(conversationMessages).where(eq(conversationMessages.conversationId, id))` then `db.delete(conversations).where(eq(conversations.id, id))`
  5. Return `{ success: true }`
  6. Import `conversationMessages` from schema if not already imported
