3. Watch the "Deploy to Production" workflow run
4. After it completes (~2-3 min), visit https://zap.iqui27.app

**If workflow fails:**
- Click the failed run → expand the "Deploy via SSH" step
- Common issues:
  - "Permission denied" → SSH key format wrong (must include BEGIN/END headers)
  - "pm2: command not found" → add `export PATH=$PATH:/usr/local/bin:/root/.nvm/versions/node/*/bin` to deploy.sh
  - "Build failed" → check Node.js version on server (`node -v`, needs 18+)
