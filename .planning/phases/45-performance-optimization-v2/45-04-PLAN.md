Purpose: Derived state should be computed during render, not in effects. Effects are for side effects (fetching, subscriptions), not for synchronizing state within a component.

Output: 10-15 instances of unnecessary useEffect removed, replaced with direct computation.
</objective>

  </behavior>
  <action>
    Fix derived state in CRM:
    
    **Common anti-patterns:**
    ```typescript
    // ❌ Don't do this
    const [filteredVoters, setFilteredVoters] = useState([]);
    
    useEffect(() => {
      let result = voters;
      if (search) result = result.filter(v => v.name.includes(search));
      if (tagFilter) result = result.filter(v => v.tags.includes(tagFilter));
      setFilteredVoters(result);
    }, [voters, search, tagFilter]);
    
    // ✅ Do this instead
    const filteredVoters = useMemo(() => {
      let result = voters || [];
      if (search) result = result.filter(v => v.name.includes(search));
      if (tagFilter) result = result.filter(v => v.tags.includes(tagFilter));
      return result;
    }, [voters, search, tagFilter]);
    ```
    
    Note: Use `useMemo` for expensive computations, but many filters are fast enough to compute without memoization.
    
    For simple filters:
    ```typescript
    const filteredVoters = (voters || [])
      .filter(v => !search || v.name.includes(search))
      .filter(v => !tagFilter || v.tags.includes(tagFilter));
    ```
  </action>
  <verify>
    <automated>
      ! grep -q "useEffect.*setFiltered" src/app/crm/page.tsx
    </automated>
  </verify>
  <done>
    - No useEffect for filtered voters
    - Filters computed during render
    - Count statistics computed directly
  </done>
