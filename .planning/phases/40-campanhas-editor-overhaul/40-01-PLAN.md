  1. **Gemini message generation functions** — Add to src/lib/gemini.ts:

     ```typescript
     // Generate a new campaign message from a prompt
     export async function generateMessage(params: {
       prompt: string;              // "Convite para reunião no bairro Centro"
       tone?: 'formal' | 'informal' | 'friendly' | 'urgent'; // default: 'informal'
       maxLength?: number;          // target char count (default: 500)
       candidateName?: string;      // for personalization context
       segmentDescription?: string; // "Moradores do bairro Centro, 25-45 anos"
       includeVariables?: boolean;  // include {nome}, {bairro} etc. (default: true)
     }): Promise<{ message: string; suggestions: string[] }>

     // Improve an existing message (fix grammar, improve tone, add persuasion)
     export async function improveMessage(params: {
       message: string;             // current message text
       instruction?: string;        // "Torne mais persuasivo" or "Adicione urgência"
       candidateName?: string;
     }): Promise<{ improved: string; changes: string[] }>

     // Rewrite message in a completely different style
     export async function rewriteMessage(params: {
       message: string;
       style: 'shorter' | 'longer' | 'more_formal' | 'more_casual' | 'more_persuasive';
       candidateName?: string;
     }): Promise<{ rewritten: string }>
     ```

     **System prompt for all message generation:**
     ```
     Você é um especialista em marketing político eleitoral via WhatsApp.
     Regras:
     - Mensagens devem ser em português brasileiro
     - Use linguagem direta e pessoal
     - Inclua variáveis de personalização quando apropriado: {nome}, {bairro}, {candidato}
     - Respeite o limite de caracteres solicitado
     - Mensagens devem seguir boas práticas de WhatsApp (parágrafos curtos, sem muita formatação)
     - Nunca inclua conteúdo ofensivo, fake news, ou promessas ilegais
     - Use formatação WhatsApp quando apropriado: *bold* para destaques, _italic_ para ênfase
     ```

     Use the existing gemini model instance (gemini-2.5-flash). Log via syslog. Handle errors gracefully.

  2. **API endpoint** — Create src/app/api/gemini/generate/route.ts:
     ```
     POST /api/gemini/generate
     Body: { action: 'generate' | 'improve' | 'rewrite', ...params }
     Response: { result: { message/improved/rewritten, suggestions/changes } }
     ```
     - Require authentication (use existing session check)
     - Validate input (prompt required for generate, message required for improve/rewrite)
     - Rate limit: max 10 calls per minute per session (simple in-memory counter)
     - Log via syslog: `syslog.info('gemini', 'Message generation requested', { action })`
