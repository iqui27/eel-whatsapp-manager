# Phase 07 Plan 01: Compliance (LGPD)

**Phase:** 07-compliance-admin
**Plan:** 07-01
**Type:** autonomous
**Goal:** LGPD compliance page — opt-in status cards, consent management table with search, anonymization trigger, and audit timeline export — LGPD-01 + LGPD-02

---

## Context

- `consentLogs` table: id, voterId, action (opt_in/opt_out/revoke), channel, metadata, createdAt
- `voters` table: optInStatus (active/pending/expired/revoked), optInDate
- `/api/compliance` exists: GET with ?voterId= filter
- `db-compliance.ts` has: `logConsent`, `getConsentStats`, `getVoterConsentHistory`
- `/api/voters` supports GET with search + PUT for updates

---

## Tasks

### Task 1 — Update compliance API

Extend `src/app/api/compliance/route.ts`:
- **GET**: support `?voterId=` (already done) + `?action=` filter + no params → return all logs with voter name join (from db-voters lookup)
- **POST**: accept `{ voterId, action, channel, metadata }` → call `logConsent()` from db-compliance.ts
- Stats endpoint at same route: GET with `?stats=1` → return `{ total, active, pending, expired, revoked }` counts

### Task 2 — Compliance page (`/conformidade/page.tsx`)

Create `src/app/conformidade/page.tsx`:
- **Header**: "Conformidade LGPD" + "Exportar relatório" button (downloads consent_logs as CSV)
- **Status cards row** (4 cards): Ativos, Pendentes, Expirados, Revogados — each with count + icon + color
- **Consent table**: voter name, phone, opt-in status badge, consent date, last action, Actions
  - Searchable by voter name/phone (debounced)
  - Filter by status (dropdown)
  - Each row: "Ver histórico" (expands inline), "Revogar" (sets status=revoked), "Anonimizar" (masked name/phone)
- **Anonymize action**: PUT /api/voters with `{ name: "Eleitor Anônimo", phone: "****", cpf: "***" }` + optimistic update

### Task 3 — Audit timeline (inside compliance page)

Below consent table:
- Tab: "Linha do Tempo de Auditoria"
- Fetch all consent logs (GET /api/compliance with no filters)
- Timeline list: each entry = icon + "voter did action via channel" + date
- Filter by: voter name search, action type (opt_in/opt_out/revoke), date range (last 7d / 30d / all)
- "Exportar CSV" button: generates and downloads as `auditoria-lgpd-YYYY-MM-DD.csv`

### Task 4 — Update sidebar nav + TypeScript check

In `src/components/SidebarLayout.tsx`:
- Ensure "Conformidade" nav item points to `/conformidade` (check if already present, add if not)

Run TypeScript check (0 errors required).
