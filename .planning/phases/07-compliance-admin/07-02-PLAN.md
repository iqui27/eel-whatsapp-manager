# Phase 07 Plan 02: Admin (Users + Permissions)

**Phase:** 07-compliance-admin
**Plan:** 07-02
**Type:** autonomous
**Goal:** Admin panel — user management (list, invite, role assignment, remove) and permissions matrix view — ADM-01 + ADM-02

---

## Context

- `users` table: id, email, name, role (admin/coordinator/volunteer), region, active, createdAt
- `db-users.ts`: loadUsers, addUser, updateUser, deleteUser
- `/api/users` doesn't exist yet — must be created
- Roles in schema: `admin`, `coordinator`, `volunteer`

---

## Tasks

### Task 1 — Users API

Create `src/app/api/users/route.ts`:
- **GET**: return all users (no auth for MVP) or filtered by `?role=`
- **POST**: create user `{ email, name, role, region }` → `addUser()`
- **PUT**: update user `{ id, ...fields }` → `updateUser()` 
- **DELETE**: `?id=` → `deleteUser()`

### Task 2 — Admin page (`/admin/page.tsx`)

Create `src/app/admin/page.tsx`:
- **Header**: "Administração" + "Convidar usuário" button → opens dialog
- **Users table**:
  - Columns: Name, Email, Role badge (Admin/Coordenador/Voluntário), Region, Status (Ativo/Inativo), Actions
  - Role badge colors: Admin=purple, Coordenador=blue, Voluntário=green
  - Actions: "Editar função" (inline role dropdown), "Remover" (confirm dialog)
- **Invite dialog** (Dialog from shadcn):
  - Fields: Name, Email, Role (select), Region (text input)
  - Submit → POST /api/users
  - Optimistic insert, refresh table on success

### Task 3 — Permission matrix (inside admin page)

Second tab "Permissões":
- Static matrix table showing module vs role permissions
- Rows = modules: Dashboard, CRM, Campanhas, Conversas, Conformidade, Admin
- Columns = roles: Admin, Coordenador, Voluntário
- Cells = ✓/✗ icons (static — no DB backing, purely informational for MVP)
- Note below table: "Permissões personalizadas por usuário disponíveis em breve"

### Task 4 — Update sidebar + TypeScript check + build

- Check `src/components/SidebarLayout.tsx` has "Admin" nav item pointing to `/admin`
- Run `node_modules/.bin/tsc --noEmit` (0 errors)
- Run `npm run build` (passes cleanly)
