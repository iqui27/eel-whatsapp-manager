# Phase 17 Plan 01: Webhook Handlers for Delivery Tracking

## Objective

Implement complete delivery lifecycle tracking via MESSAGES_UPDATE webhook, reply correlation to campaigns, and group join conversion tracking.

## Context

- Message queue exists from Phase 15 with 8-state lifecycle
- Evolution API sends MESSAGES_UPDATE events with status codes
- GROUP_PARTICIPANTS_UPDATE already handled for size tracking (Phase 16)
- Need to correlate replies to campaigns and track conversions

### Task 2: Update MESSAGES_UPDATE webhook handler
type: auto

**File:** `src/app/api/webhook/route.ts`

**Update the placeholder handler:**

Parse status codes:
- 0 = error
- 1 = pending
- 2 = server_ack (sent)
- 3 = delivered
- 4 = read
- 5 = played

**Logic:**
1. Get message key (remoteJid, id)
2. Find message in message_queue by evolutionMessageId
3. Update status and timestamps:
   - status=3 → delivered, set deliveredAt
   - status=4 → read, set readAt
   - status=0 → failed, set failedAt, failReason
4. Increment campaign counters (totalDelivered, totalRead)

**Verification:** Handler processes status updates correctly.

### Task 3: Implement reply correlation
type: auto

**File:** `src/app/api/webhook/route.ts`

**In messages.upsert handler:**

When receiving inbound message:
1. Extract sender phone (remoteJid)
2. Find recent messages in queue sent to this phone
3. If found and campaign is active:
   - Check if this is a reply (within 7 days of send)
   - Increment campaign.totalReplied
   - Create conversion_event with type='reply'

**Verification:** Replies tracked and counted.

### Task 4: Implement group join conversion tracking
type: auto

**File:** `src/app/api/webhook/route.ts`

**In group_participants.update handler:**

When action='add':
1. Get participant phone(s)
2. For each participant:
   - Find recent messages containing invite link
   - Match to campaign via group's campaignId
   - Increment campaign.totalJoinedGroup
   - Create conversion_event with type='group_join'

**Verification:** Group joins tracked as conversions.

### Task 6: Create migration SQL
type: auto

**File:** `drizzle/0008_conversion_events.sql`

**Content:**
```sql
-- Add conversion fields to campaigns
ALTER TABLE campaigns ADD COLUMN IF NOT EXISTS total_clicked INTEGER DEFAULT 0;
ALTER TABLE campaigns ADD COLUMN IF NOT EXISTS total_joined_group INTEGER DEFAULT 0;

-- Create conversion_events table
CREATE TABLE IF NOT EXISTS conversion_events (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  campaign_id UUID NOT NULL REFERENCES campaigns(id) ON DELETE CASCADE,
  voter_id UUID REFERENCES voters(id) ON DELETE SET NULL,
  voter_phone TEXT,
  event_type TEXT NOT NULL CHECK (event_type IN ('reply', 'click', 'group_join', 'conversion')),
  group_jid TEXT,
  metadata JSONB,
  created_at TIMESTAMPTZ DEFAULT now()
);

CREATE INDEX IF NOT EXISTS idx_conversion_events_campaign ON conversion_events(campaign_id);
CREATE INDEX IF NOT EXISTS idx_conversion_events_type ON conversion_events(event_type);
CREATE INDEX IF NOT EXISTS idx_conversion_events_created ON conversion_events(created_at);
```
