<objective>
Add an in-memory session cache to db-auth.ts to eliminate 1-2 database round trips per authenticated API request.

Purpose: Every authenticated request currently calls `getStoredSession()` (1 DB query) and then `getUser()` (1 more DB query). With ~20+ API calls per page load and 6 polling endpoints, this generates massive unnecessary DB load. A 60-second in-memory cache eliminates >95% of these queries.
Output: Cached session lookups, reduced DB connection pool pressure, faster API response times.
</objective>

Modify `src/lib/db-auth.ts` to add an in-memory session cache with 60-second TTL. The changes:

1. **Add cache data structure** at module level (after imports):
   ```ts
   interface CachedSession {
     actor: SessionActor;
     cachedAt: number;
   }
   const SESSION_CACHE_TTL_MS = 60_000; // 60 seconds
   const SESSION_CACHE_MAX_SIZE = 500;  // max cached sessions
   const sessionCache = new Map<string, CachedSession>();
   ```

2. **Modify `getSessionActor()`** to check cache first:
   - Before hitting DB, check `sessionCache.get(token)`
   - If found AND `Date.now() - entry.cachedAt < SESSION_CACHE_TTL_MS`, return the cached `actor` immediately (zero DB queries)
   - If not found or expired, proceed with existing DB logic
   - After successful DB lookup, store result in cache: `sessionCache.set(token, { actor, cachedAt: Date.now() })`
   - Before inserting, if `sessionCache.size >= SESSION_CACHE_MAX_SIZE`, evict oldest 100 entries (iterate Map in insertion order, delete first 100)

3. **Modify `destroySession()`** to invalidate cache:
   - Add `sessionCache.delete(token)` before the DB delete call
   - This ensures logout is immediate even with cache

4. **Move expired session purge from `createSession()`** to a lazy approach:
   - Remove the `await db.delete(sessions).where(lt(sessions.expiresAt, new Date()))` from `createSession()`
   - Add a module-level variable `let lastPurgeAt = 0;` and a constant `const PURGE_INTERVAL_MS = 5 * 60_000; // 5 min`
   - In `getStoredSession()`, before the DB query, check if `Date.now() - lastPurgeAt > PURGE_INTERVAL_MS`. If so, fire-and-forget the purge (don't await it):
     ```ts
     if (Date.now() - lastPurgeAt > PURGE_INTERVAL_MS) {
       lastPurgeAt = Date.now();
       db.delete(sessions).where(lt(sessions.expiresAt, new Date())).catch(() => {});
     }
     ```
   - This avoids the purge penalty on every login and moves it to a lazy background task.

5. **Keep all existing function signatures unchanged** — this is a drop-in enhancement, no API contract changes.
