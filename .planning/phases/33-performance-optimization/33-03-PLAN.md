For EACH of the 6 files with setInterval polling, add a `document.visibilityState` guard. The pattern is the same for all — wrap the existing setInterval logic:

**Pattern to apply in each file's polling useEffect:**
```tsx
useEffect(() => {
  // ... existing setup ...
  
  let intervalId: ReturnType<typeof setInterval> | null = null;
  
  const startPolling = () => {
    if (intervalId) return;
    intervalId = setInterval(() => {
      if (document.hidden) return; // skip fetch when tab is hidden
      fetchFunction();
    }, INTERVAL_MS);
  };
  
  const stopPolling = () => {
    if (intervalId) {
      clearInterval(intervalId);
      intervalId = null;
    }
  };
  
  const handleVisibilityChange = () => {
    if (document.hidden) {
      stopPolling();
    } else {
      fetchFunction(); // immediate refresh when tab becomes visible
      startPolling();
    }
  };
  
  document.addEventListener('visibilitychange', handleVisibilityChange);
  startPolling();
  
  return () => {
    document.removeEventListener('visibilitychange', handleVisibilityChange);
    stopPolling();
  };
}, [dependencies]);
```

Three targeted optimizations to `src/app/api/conversations/stream/route.ts`:

1. **Increase poll interval from 1.5s to 5s:**
   Change `const POLL_INTERVAL_MS = 1500;` → `const POLL_INTERVAL_MS = 5000;`
   This reduces DB queries from ~40/min to ~12/min per SSE connection. The heartbeat is already 15s so this is still responsive for chat.

2. **Add per-cycle voter lookup cache** to eliminate N+1 queries:
   Inside the `run()` async function, before the while loop, create a Map:
   ```ts
   const voterCache = new Map<string, Awaited<ReturnType<typeof getVoter>>>();
   ```
   
   Replace the two `await getVoter(...)` calls (lines ~95 and ~128-129) with a cached version:
   ```ts
   async function getCachedVoter(id: string) {
     if (voterCache.has(id)) return voterCache.get(id)!;
     const voter = await getVoter(id);
     voterCache.set(id, voter);
     return voter;
   }
   ```
   
   Replace `await getVoter(conversation.voterId)` on line ~95 with `await getCachedVoter(conversation.voterId)`.
   Replace `await getVoter(filters.voterId)` on line ~128 with `await getCachedVoter(filters.voterId)`.
   
   Clear the cache periodically (every 60s or 12 cycles) to avoid stale data:
   ```ts
   let cycleCount = 0;
   // Inside the while loop, after all processing:
   cycleCount++;
   if (cycleCount % 12 === 0) voterCache.clear(); // refresh voter data every ~60s
   ```

3. **Cap trackedConversationIds to prevent unbounded growth:**
   Add a max size constant: `const MAX_TRACKED_CONVERSATIONS = 200;`
   Before adding to the set, check size:
   ```ts
   if (trackedConversationIds.size < MAX_TRACKED_CONVERSATIONS) {
     trackedConversationIds.add(conversation.id);
   }
   ```
   This prevents memory leaks from long-running SSE connections that accumulate conversation IDs.
